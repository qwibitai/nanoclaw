# aix-explainer

Deliver a crisp, audience-tailored explanation of the **AI+X** approach.

AI+X framing: **AI amplifies a specific domain (X)** by combining:
- domain goals + constraints (what "good" means in X)
- domain workflows (where decisions happen)
- domain data + feedback loops (how the system improves)
- AI capabilities (models, tools, automation) applied *safely* and *measurably*

## Workflow

### 1) Identify the audience and desired outcome
Ask 1-2 questions only:
- "Who am I explaining this to (role)?"
- "Do they want a concept overview, a project plan, or a product pitch?"

Then choose an output mode:
- **30-second elevator pitch**
- **2-minute explanation**
- **5-minute structured talk track**
- **FAQ / objections handling**
- **One-pager outline**

### 2) Use the AI+X talk track (default)
Use this structure unless user requests otherwise:
1. **Problem**: X has high-cost decisions / repetitive work / knowledge bottlenecks.
2. **Why not "pure AI"**: without X, you get demos that don't integrate, don't measure, don't govern.
3. **Definition**: AI+X = domain system + AI augmentation.
4. **Where AI fits** (3 buckets):
   - Assist (draft/summarize/answer)
   - Automate (workflow steps, routing, extraction)
   - Optimize (decision support, prediction, experimentation)
5. **How it succeeds** (pillars):
   - measurable outcomes (KPIs)
   - data flywheel (feedback + evaluation)
   - human-in-the-loop + governance
   - integration into existing tools/processes
6. **Example**: 1 concrete example with before/after.
7. **Next step**: suggest a minimum viable pilot.

### 3) Always include "minimum viable pilot"
Propose a pilot that is:
- 2-6 weeks
- narrow scope
- clear metric
- uses existing data
- includes evaluation + guardrails

### 4) Provide 2-3 examples across domains (optional)
If the audience is general, pick examples from:
- Education (teacher companion / tutoring / grading assist)
- Standards/compliance (document diff, requirements traceability)
- Business operations (customer support, sales enablement)
- Engineering (code review, incident triage)

### 5) Handle objections (if asked)
Use the FAQ in `references/faq.md`.

## Output templates

### A) 30-second pitch (template)
"AI+X means taking the *real workflow* in **X** and using AI to remove bottlenecks--without breaking the rules of the domain. Instead of a generic chatbot, you get a system that knows the goals, constraints, and metrics of X, plugs into existing processes, and improves through feedback. The result is measurable outcomes: faster decisions, higher quality, and lower cost--safely."

### B) 2-minute explanation (template)
Use: definition -> why -> where AI fits -> what makes it work -> example -> pilot.

### C) One-pager outline
- Context / problem
- AI+X definition
- Use cases (3)
- Architecture sketch (data -> model -> workflow -> evaluation)
- Governance & safety
- Pilot plan
