# 🏗️ News Briefing System - Technical Architecture

## Overview

The News Briefing System is **built on top of nanoclaw** and integrates deeply with its infrastructure for scheduling, WhatsApp delivery, and agent coordination.

---

## 🔗 Integration with Nanoclaw

### 1. Scheduling (Cron)

**How it works:**
- Uses nanoclaw's `schedule_task` MCP tool
- Cron expression: `0 7 * * *` (7:00 AM daily)
- Context mode: `group` (access to memory and conversation history)

**Where it's configured:**
```
Task ID: 1773071284969-xq6gr8.json
Located in: nanoclaw's task scheduler
Command: python3 /workspace/group/news-briefing-poc/main.py
```

**View scheduled tasks:**
```bash
# Use nanoclaw MCP tool
mcp__nanoclaw__list_tasks
```

---

### 2. WhatsApp Delivery (IPC)

**How it works:**
- System generates PDF in `/workspace/group/news-briefing-poc/reports/`
- Creates IPC message with file path
- Writes to `/workspace/ipc/messages/`
- Nanoclaw's IPC watcher picks it up
- WhatsApp channel sends the PDF

**IPC Message Format:**
```json
{
  "type": "file",
  "chatJid": "15415134857@s.whatsapp.net",
  "file_path": "/workspace/group/news-briefing-poc/reports/briefing_2026-03-09.pdf",
  "caption": "📰 Your Daily News Briefing...",
  "groupFolder": "main",
  "timestamp": "2026-03-09T..."
}
```

**IPC Flow:**
```
main.py → Creates IPC JSON → /workspace/ipc/messages/
                                       ↓
                            Nanoclaw IPC Watcher
                                       ↓
                            WhatsApp Channel (Baileys)
                                       ↓
                            PDF delivered to user
```

---

### 3. Agent Research

**Current PoC Implementation:**
- Uses Claude's `Task` tool to spawn agents
- 4 agents run in parallel (haiku model for speed)
- Each agent uses `WebSearch` to find news
- Results saved to JSON files

**Agent Flow:**
```
Orchestrator creates tasks
        ↓
Spawns 4 parallel Task agents
        ↓
Agent 1: World    Agent 2: Tech    Agent 3: Finance    Agent 4: Custom
   (Haiku)           (Haiku)           (Haiku)            (Haiku)
        ↓                ↓                ↓                  ↓
   WebSearch        WebSearch        WebSearch          WebSearch
        ↓                ↓                ↓                  ↓
   JSON result      JSON result      JSON result        JSON result
        └────────────────┴────────────────┴──────────────────┘
                                ↓
                        Compiler collects all
```

**Production Enhancement:**
- Would use `TeamCreate` for true parallel execution
- Current version spawns sequentially (PoC simplification)

## 🧠 Memory System

### Purpose
Prevent showing the same articles multiple times across days.

### How It Works

**Article Hashing:**
```python
hash = MD5(title + "|" + url)
# Example: "abc123def456..."
```

**Memory Structure:**
```json
{
  "last_briefing_date": "2026-03-09",
  "seen_articles": [
    "abc123def456...",  // Hash of article 1
    "def456ghi789...",  // Hash of article 2
    // ... up to 500 hashes
  ],
  "topic_history": {
    "world_highlights": [
      {
        "date": "2026-03-09",
        "article_count": 5,
        "trends": ["Middle East conflict", "Energy crisis"]
      }
    ]
  }
}
```

**Deduplication Logic:**
```
For each new article:
  1. Generate hash from (title + URL)
  2. Check if hash exists in memory
  3. If YES → Skip (duplicate)
  4. If NO → Include in briefing
  5. Add hash to memory
```

**Memory Limits:**
- Keeps last **500 article hashes** (~30 days)
- Keeps last **30 entries** per topic in history
- Auto-cleanup on each run

## 🐛 Troubleshooting

### "0 articles" Problem

**Cause:** All articles are in memory (already seen)

**Why it happens:**
1. System ran earlier today
2. Researched 20 articles
3. Stored hashes in memory
4. Second run finds same articles → all filtered as duplicates

**Solutions:**
```bash
# Option 1: Clear memory (for testing)
rm /workspace/group/news-briefing-poc/memory/briefing_memory.json

# Option 2: Wait for tomorrow (new news will be published)

# Option 3: Change topics (to get different content)
python3 topic_manager.py add technology "new specific topic"
```

### PDF Not Delivered

**Check 1: IPC message created?**
```bash
ls /workspace/ipc/messages/
# Should be empty (processed) or show briefing_*.json (pending)
```

**Check 2: Nanoclaw logs**
```bash
tail -50 /workspace/project/logs/nanoclaw.log | grep "Document sent"
# Should show "Document sent" and "IPC file sent"
```

**Check 3: PDF exists locally?**
```bash
ls /workspace/group/news-briefing-poc/reports/briefing_*.pdf
# PDF should be there even if delivery failed
```

### Agent Research Fails

**Check WebSearch availability:**
- Agents need WebSearch tool to function
- If WebSearch fails, agents return empty results

**Check rate limits:**
- Multiple rapid runs might hit search rate limits
- Wait a few minutes between runs
