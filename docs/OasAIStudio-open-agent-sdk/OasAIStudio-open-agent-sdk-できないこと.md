コードを調査した結果、具体的に動かない・制限のある箇所は以下の通りです。

---

## 1. MCP の SSE / SDK トランスポートが未実装

`McpManager.connect()` で `type: 'sse'` または `type: 'sdk'` を指定すると、即座に例外が投げられます。 [1](#0-0) 

テストでも「not yet implemented」として明示的に確認されています。 [2](#0-1) 

---

## 2. `FileStorage` は Bun 専用（Node.js では動かない）

`FileStorage` の実装全体が `Bun.file()`、`Bun.write()`、`Bun.spawn()` に依存しており、Node.js 環境では動作しません。 [3](#0-2) 

---

## 3. `FileStorage` でカスタムセッション ID が使えない

UUID 形式以外のセッション ID を渡すと明示的にエラーになります（v0.2.0 時点で非対応と明記）。 [4](#0-3) 

---

## 4. `autoCompactThreshold` / `preserveRecentRounds` が公開 API から設定できない

`ReActLoopConfig` にはこれらのオプションが存在しますが、`CreateSessionOptions` には含まれておらず、`createSession()` / `resumeSession()` / `forkSession()` のループ生成時にも渡されていません。 [5](#0-4) [6](#0-5) 

`CreateSessionOptions` にこれらのフィールドがないため、ユーザーが設定する手段がありません。

---

## 5. `forkSession` で `outputFormat` が引き継がれない

`forkSession()` 内で `ReActLoop` を生成する際、`outputFormat` が渡されていません。フォーク元のセッションが構造化出力を使っていても、フォーク後のセッションでは無効になります。 [7](#0-6) 

---

## 6. `enableFileCheckpointing` + `HookManager` インスタンスの組み合わせで既存フックが消える

`hooks` に `HookManager` インスタンスを渡した状態で `enableFileCheckpointing: true` にすると、`mergeHooks()` 内で既存フックの内容が取り出せず、チェックポイントフックだけが残ります（既存フックはサイレントに消える）。 [8](#0-7) 

---

## 7. `AskUserQuestion` ツールが `canUseTool` なしでは機能しない

`canUseTool` コールバックが設定されていない場合、`AskUserQuestion` はエラーメッセージを返すだけで、ユーザーへの質問が行われません。 [9](#0-8) 

---

## 8. `clearCheckpoints` がペンディング中のチェックポイントを消せない

`clearCheckpoints()` はコミット済みのチェックポイントしか削除せず、実行中のツールに対応するペンディング状態のチェックポイントは残ります。コード内にもその旨のコメントがあります。 [10](#0-9) 

---

**まとめ**: 即座にエラーになるのは MCP の SSE/SDK トランスポートと非 UUID セッション ID。サイレントに壊れるのは `autoCompactThreshold` の設定不能、`forkSession` の `outputFormat` 欠落、`HookManager` + チェックポイントの組み合わせです。
