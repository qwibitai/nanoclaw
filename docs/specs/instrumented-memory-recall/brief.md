# Brief: Instrumented Memory Recall Improvements

> Generated by `/team-brief` — 2026-05-06
> Saved to: `docs/specs/instrumented-memory-recall/brief.md`

## Context

NanoClaw's host daemon currently injects mnemon recall results into agent prompts before each turn via a `[Recalled context]` system message. Recall is wired end-to-end (`src/modules/memory/recall-injection.ts` → `src/modules/memory/mnemon-impl.ts` → `mnemon recall` CLI), but **we have zero ground truth on whether it helps**. We see "facts returned" but not "facts useful." The query passed to mnemon is an 800-char raw smear of the last three user messages — exactly the input shape upstream's own skill says to avoid. We are optimizing blind.

This feature instruments recall first, then improves it on the basis of measured signal, then opens cross-group recall as a third deliverable. Three tightly-coupled deliverables shipped as one feature because each later step is gated on the measurement substrate the first step provides.

## Requirements

### R1 — Recall feedback loop (measurement substrate)
- **R1.1** New `recall_outcomes` table in `data/mnemon-ingest.db` capturing `(turn_id, fact_ids_returned, query_strategy, judge_score_per_fact, judge_method, embedding_sim_per_fact, judged_at)`.
- **R1.2** Async post-turn scoring step: after the agent's response is finalized, score each recalled fact against the response. Two signals — cheap (embedding cosine similarity) and expensive (LLM judge with a 0/1/2 rubric: ignored / partial / load-bearing).
- **R1.3** LLM judge backend reuses the existing classifier drop-in mechanism (`memory-daemon-backend.conf`); default Anthropic Haiku 4.5; operator can switch to Codex/Sonnet/Opus per existing pattern.
- **R1.4** Judge calls follow the existing classifier failure pattern (retry, dead-letter on persistent failure). Coverage target: ≥95% of recall events get a score within 24h.
- **R1.5** Eval harness: ~50 query/expected-fact pairs synthesized from real group stores. Used as the cheap pre-flight gate before live A/B. Regenerated quarterly or via `scripts/regenerate-recall-eval.ts`.
- **R1.6** Sampling strategy: always-on judge for the first 200 events per group (calibration phase). Once Spearman ρ ≥ 0.5 holds between cheap signal and judge for that group, drop to 1-in-5 sampling. If correlation never reaches threshold, stay always-on indefinitely.
- **R1.7** Per-group rollback: new `memory.feedback_enabled: boolean` field in `groups/<g>/container.json` (defaults to true when memory is enabled). Disabling stops new judging; existing rows preserved.
- **R1.8** Retention: 90-day rolling window for `recall_outcomes` rows; daemon prunes nightly. Aggregated metrics persist in `data/memory-health.json` indefinitely.
- **R1.9** Operator-visible health: extend `data/memory-health.json` per-group with `recall_quality: { coverage_24h, mean_judge_score_7d, useful_fact_rate_7d, spearman_rho_calibration, sampling_mode }`.

### R2 — Query improvement (Strategy C)
- **R2.1** Replace the raw 3-turn-smear in `extractRecallQueryText` (`src/modules/memory/recall-injection.ts:168-183`) with an LLM-extracted focused keyword query.
- **R2.2** Three-tier fallback: LLM extractor (800ms timeout) → Strategy B heuristic (stopword strip + entity-prefer, ~80 chars) → Strategy A current behavior (raw smear). Recall must never break because of an extractor failure.
- **R2.3** Cache extracted queries by 3-turn-hash; same conversation slice never re-extracts.
- **R2.4** Per-group config flag `memory.query_strategy: "raw" | "heuristic" | "llm"`; defaults to `"raw"` for parity with current behavior. Operator flips per group only after the live A/B passes its gate.

### R3 — Cross-group recall scope
- **R3.1** Per-group config field `memory.recall_scope: "self" | "all-groups" | string[]` in `groups/<g>/container.json`. Default `"self"` preserves current isolation.
- **R3.2** Daemon-side fan-out: when `recall_scope` includes other groups, daemon issues parallel `mnemon recall` calls across the relevant stores, merges results, deduplicates by fact id, and re-ranks before injection.
- **R3.3** Failure path: if cross-group fan-out's mean judge score drops below per-group `self`-scope baseline by ≥0.2 (0-2 scale) for 7 consecutive days, daemon **auto-reverts** that group to `self` AND writes `recall_scope_reverted: { group, from, to, reason, at }` to `data/memory-health.json`. Circuit breaker does not auto-flip back; operator re-enables manually.
- **R3.4** This feature ships only the mechanism. No group's `recall_scope` is changed by the feature itself. Operator flips axis-labs to `"all-groups"` separately when ready.

## Constraints

- **[HARD] C1** — Daemon is the sole writer to mnemon stores. No agent-write paths introduced. Source: existing security boundary, hardened in commits a060c02, 88892f2 (Codex F-series fixes).
- **[HARD] C2** — Recall remains pre-turn host-injected via `kind=system trigger=0` row in `inbound.db`. No mid-turn agent-pulled recall path in this feature. Source: scope decision; deferred until measurement justifies it.
- **[HARD] C3** — Per-group store isolation preserved by default. Cross-group reads are explicit opt-in via `recall_scope` config; no implicit fan-out. Source: tenant isolation principle.
- **[HARD] C4** — Measurement before optimization. R2 (query improvement) and R3 (scope) cannot ship until R1 (feedback loop) is live and producing scores. Source: explicit user requirement, restated multiple times during requirements pass.
- **[HARD] C5** — Strategy C (LLM query extraction) must never break recall. On any extractor failure path, fall back through Strategy B then Strategy A. Source: reliability requirement.
- **[HARD] C6** — Cross-group fan-out must respect existing per-store mount narrowing. Daemon cannot mount additional stores into containers; fan-out happens host-side and only merged results cross the trust boundary. Source: existing tenant isolation; cannot be relaxed.
- **[HARD] C7** — Supply-chain hygiene preserved: pnpm `minimumReleaseAge: 4320` retained; no `minimumReleaseAgeExclude` additions; no new `onlyBuiltDependencies` entries. Source: project CLAUDE.md.
- **[HARD] C8** — Quality gates preserved: host typecheck (tsc), container typecheck (bun run typecheck), vitest on host, bun:test in container. Source: project quality_gates.
- **[SOFT] C9** — LLM judge cost should remain under ~$10/month at current ~11-group volume. At Haiku 4.5 prices and observed recall volume, always-on judging is well under this; sampling is for signal quality, not cost.
- **[SOFT] C10** — Eval-harness fact-pair set is stable across A/B comparisons. Manual regeneration only; no auto-regen on corpus drift.

## Non-Goals (Explicitly Out of Scope)

- **Agent-pulled MCP recall tool.** The agent already has Bash access to mnemon today via the wrapper. Wrapping it as MCP is deferred until measured signal shows daemon injection is missing context the agent could fetch on demand.
- **Upstream-style global store mount** (`~/.mnemon/data/global/` mounted into every container). Cross-group `recall_scope` replaces this with a more flexible model. The shared-mount approach is rejected; do not propose during design.
- **Adopting upstream's `/add-mnemon` skill, `container-skill.md`, or lifecycle hooks** (prime/remind/nudge/compact). These implement an agent-driven write+recall model architecturally incompatible with our daemon-write-only design. Do not propose during design.
- **Agent-side `mnemon remember` calls.** Daemon stays sole writer. The block-mnemon-real-hook plus `MNEMON_READ_ONLY=1` wrapper gate must remain in force.
- **Switching mnemon to a long-lived daemon.** Out of scope; not blocking this feature.
- **Auto-promotion of facts between stores** (e.g., "this fact appeared in 3 groups → promote to shared"). Cross-group reads make auto-promotion unnecessary.
- **Bumping mnemon binary 0.1.2 → 0.1.3.** Tracked separately; not gated on this feature.

## Key Concepts

- **Recall event:** A single invocation of `MnemonStore.recall()` from `maybeInjectRecall`, identified by `recall-${inboundMessage.id}`. The unit of measurement for the feedback loop.
- **Load-bearing fact:** A recalled fact that materially informed the agent's response (LLM judge score = 2). Distinguishes true positives from facts that were returned but ignored.
- **Useful fact rate:** % of *returned* facts (not events) that scored ≥1. Precision-style metric. Complements coverage-style "≥1 load-bearing fact returned."
- **Calibration phase:** Per-group window during which the LLM judge runs always-on for the first 200 events to fit the embedding-similarity signal against the judge. After calibration, sampling kicks in only if correlation passes threshold.
- **Strategy A / B / C:** Query construction strategies. A = current 800-char raw smear. B = stopword/entity heuristic, ~80 chars, free. C = LLM-extracted focused keyword query, ~200ms, ~$0.0001/call.
- **`recall_scope`:** Per-group config controlling which mnemon stores the daemon queries during recall injection. `"self"` = current behavior. `"all-groups"` = fan out across all enabled groups. `string[]` = explicit allowlist.
- **Circuit breaker (R3.3):** Auto-revert mechanism for `recall_scope` regressions. Triggers on 7 consecutive days of mean-judge-score regression of ≥0.2 vs `self` baseline. Does not auto-flip back on; operator re-enables manually.

## Open Questions (Forced to Defaults)

- **Judge prompt format and rubric:** Defaulting to a 0/1/2 ordinal scale (ignored/partial/load-bearing) with a per-fact judgment, batched into one call per turn. Design will pin the exact prompt; reviewers should scrutinize phrasing for grade-inflation bias and self-fulfilling prophecy bias.
- **Eval-set construction methodology:** Defaulting to Haiku-synthesized user messages from real fact corpus (one fact → one synthesized query that should retrieve it). Manual operator review of the generated set before commit. Design should consider: does the synthesizer's prior leak into the evaluator's prior?
- **Cross-store merge/rerank algorithm (R3.2):** Defaulting to per-store score normalization (z-score within store) followed by recency boost (linear decay over 90 days), then global top-N. Design owns the exact formula.
- **`recall_scope` config schema:** Defaulting to flat string field on `container.json`. Design may upgrade to richer object (`{ scope, weight_per_store }`) if reranking needs it.
- **Sampling strategy hand-off (R1.6):** Defaulting to per-group calibration. Design should consider whether a global cross-group correlation is more robust than per-group.

## Success Criteria

### Feedback loop ships when:
- ≥95% of recall events get a judge score within 24h, sustained for 7 days across all enabled groups.
- Cheap signal (embedding cosine) calibrates against LLM judge with Spearman ρ ≥ 0.5 on a 200-event per-group calibration sample.
- `data/memory-health.json` exposes per-group `recall_quality` block; operator dashboard or `/memory-health` slash command surfaces it.

### Query improvement (Strategy C) ships when (both gates required):
- **Eval gate:** Strategy C lifts the rate of "≥1 load-bearing fact returned" by ≥10pp absolute over Strategy A on the eval set, AND `useful_fact_rate` does not regress by more than 5pp.
- **Live gate:** Over a 7-day live A/B across ≥3 active groups at fixed sample size, Strategy C lifts mean LLM-judge score by ≥0.3 on the 0-2 scale, AND `useful_fact_rate` does not regress by more than 5pp.

### Cross-group scope (R3) ships when:
- Mechanism is shipped behind `recall_scope` defaulting to `"self"`. No group's scope is auto-changed.
- For axis-labs (designated trial group): when manually flipped to `"all-groups"`, mean LLM-judge score over 100 events beats its `self` baseline by ≥0.2 (0-2 scale). Tie or regression = flip reverts via circuit breaker; mechanism stays shipped for other groups.

### Whole-feature ships when:
- All three deliverables meet their respective gates.
- Decision record captures every constraint, decision, and rejected alternative.
- Existing memory tests (vitest + bun:test) pass; new tests cover: feedback-loop write path, judge call retry/dead-letter, eval-harness execution, three-tier query fallback, cross-store fan-out merge, circuit-breaker revert path.
- `docs/memory.md` updated with the new operator surface (health JSON fields, scripts, config flags).
