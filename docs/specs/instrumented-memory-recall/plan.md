### Task B1: Create host-side Ollama embedding client

**File:** `src/modules/memory/cheap-signal.ts`
**Operation:** CREATE
**Test file:** `src/modules/memory/cheap-signal.test.ts`

**Approach:**
Minimal Node `fetch`-based Ollama client used only by the host process at recall time. Exposes `computeQueryFactCosines(query, facts)` returning a Map of fact_id → cosine similarity. Reads `HOST_OLLAMA_ENDPOINT` env var ONCE at module load (per S5 / D28); validates URL parses, throws loud on bad input (mirrors `parseBackendConfig` discipline at `classifier-client.ts:71-99`). 800ms total timeout — recall path has 3000ms budget; cheap signal must fit comfortably. On any failure (parse, network, timeout, malformed response), returns empty Map — recall path treats `embedding_sim=NULL` as "missing data," does NOT block.

**Interface / signature:**
```typescript
export interface FactInput {
  id: string;
  content: string;
}

export async function computeQueryFactCosines(
  query: string,
  facts: FactInput[],
  opts?: { signal?: AbortSignal; timeoutMs?: number }  // defaults to 800ms
): Promise<Map<string, number>>;
//   Returns Map<fact_id, cosine_similarity in [0, 1]>.
//   Returns empty Map on any failure (NEVER throws).
//   Empty input array → returns empty Map immediately (no Ollama call).

// Test seam (S12):
export function setEmbedderForTest(
  fn: ((texts: string[]) => Promise<number[][]>) | null
): void;
export function _resetEmbedderForTest(): void;
```

ASSERT: `HOST_OLLAMA_ENDPOINT` is read ONCE at module load via a module-level constant — NOT on every call (S5 — performance + parse-throw-loud).
ASSERT: invalid `HOST_OLLAMA_ENDPOINT` (non-URL, malformed) throws at module load, NOT at first call (parse-throw-loud per C14).
ASSERT: default endpoint when env unset: `http://127.0.0.1:11434` (D28 — different from container's `host.docker.internal`).
ASSERT: empty `facts` array returns empty Map without making any HTTP call (no wasted cost).
ASSERT: 800ms total timeout — if Ollama doesn't respond, returns empty Map (NEVER throws to caller).
ASSERT: cosine similarity is normalized to `[0, 1]` (Ollama returns vectors; cosine of normalized embeddings is in `[-1, 1]`; clip-and-rescale).
ASSERT: batched embed call — single POST to `/api/embed` with `{ model, input: [query, ...facts.map(f => f.content)] }` (one HTTP call, not N+1).

### Task C3: Create daemon judge sweep processor

**File:** `src/memory-daemon/recall-judge/judge.ts`
**Operation:** CREATE
**Test file:** `src/memory-daemon/recall-judge/judge.test.ts`

**Approach:**
Daemon-side processor invoked from `runSweep`. For each agent group with `feedback_enabled` (Task A4 resolver):
1. Query pending rows: `SELECT * FROM recall_outcomes WHERE judged_at IS NULL AND created_at < datetime('now', '-60 seconds') AND agent_group_id = ?`
2. Group by `recall_event_id` (one judge call per turn, scores all facts in one pass).
3. **Ambiguity check (M1):** if any other `recall_event_id` exists in same `(agent_group_id, trigger_thread_id)` within `[this.created_at, this.created_at + 60s]`, mark all rows in the event with `judge_method='ambiguous-correlation', judged_at=now()` and skip judging.
4. Look up agent response from `archive.db`: `SELECT text FROM messages_archive WHERE agent_group_id=? AND thread_id=? AND sent_at > ? AND role='assistant' ORDER BY sent_at LIMIT 1`. If no response yet (>60s grace already passed), retry via dead-letter (3 attempts with 60/300/900s backoff). After 3 failures, mark all rows `judge_method='judge-failed', judged_at=now()` and dead-letter the event.
5. Build judge user prompt: `{user_message, agent_response_excerpt, candidate_facts}` (response excerpt = first 2000 + last 2000 chars per S4).
6. Call `callJudge` (C1) with 30s timeout.
7. Strict-validate output: drop unmatched fact_ids, verify `score ∈ {0,1,2}`. Update each row in single transaction: `UPDATE recall_outcomes SET judge_score=?, judge_method='llm', judge_model=?, judge_evidence=?, response_excerpt_sha=?, judged_at=now() WHERE recall_event_id=? AND fact_id=? AND judge_prompt_version=?`.
8. On `JudgeParseError` or backend error: dead-letter via `recordOrIncrementFailure` with `item_type='recall-judge'`. After 3 attempts → mark `judge_method='judge-failed'`.

**Approach:**

async function pMap<T, R>(items: T[], mapper: (item: T) => Promise<R>, concurrency: number): Promise<R[]> {
  const results: R[] = new Array(items.length);
  const queue = items.map((item, idx) => ({ item, idx }));
  const workers = Array.from({ length: Math.min(concurrency, queue.length) }, async () => {
    while (queue.length > 0) {
      const next = queue.shift();

      results[next.idx] = await mapper(next.item);
