## MemoryThat Moves With You

From OpenClaw to Claude Code to Cursor to whatever's next, your own memory travels with you, not trapped in one tool.

curl -fsSL https://byterover.dev/install.sh | sh

Windows or npm? [Check the docs](https://docs.byterover.dev/)


Memory that thinks and scales

Memory that curates into a knowledge tree system

ByteRover applies stateful memory curation and a hierachical tree structure of the memory with natural language format, allowing both agents and humans to reason and manage as a knowledge system.

[Learn about ByteRover’s curate →](https://docs.byterover.dev/context-tree/curation-engine)

92.2% retrieval accuracy. Best on the market

ByteRover replaces vector-based memory with a tiered file-search retrieval pipeline, from fuzzy text → deeper LLM-driven search for higher precision.

[Read the benchmark →](https://www.byterover.dev/blog/benchmark-ai-agent-memory)


Local-first. Portable when you're ready

Everything runs on your machine by default. When you need it elsewhere - another machine, a teammate, a different agent setup -push to ByteRover's cloud. Version-controlled, editable.

[Learn about memory workspace →](https://docs.byterover.dev/context-tree/local-space-structure)


Memory shared across OpenClaw agents
With ByteRover, all of your OpenClaw agents will operate with a persistent, hierarchically structured, and shared memory.
[Explore OpenClaw setup →](https://docs.byterover.dev/autonomous-agents/openclaw)


Works with any model, any provider

You can power ByteRover with your own LLM using API key, leveraging your own existing agentic stack where you have full control over model choices, cost and observability.
