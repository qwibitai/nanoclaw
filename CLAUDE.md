# NanoClaw

Personal Claude assistant. See [README.md](README.md) for philosophy and setup. See [docs/REQUIREMENTS.md](docs/REQUIREMENTS.md) for architecture decisions.

## Quick Context

Single Node.js process with skill-based channel system. Channels (WhatsApp, Telegram, Slack, Discord, Gmail) are skills that self-register at startup. Messages route to Claude Agent SDK running in containers (Linux VMs). Each group has isolated filesystem and memory.

| File | Purpose |

## Development

Run commands directly—don't tell the user to run them.

### Worktree-based Development (Required)

This repo runs a live service with auto-update: when main changes, the process pulls, rebuilds, and restarts. To prevent incomplete feature branches from triggering auto-restart mid-development:

1. **Always use a git worktree for new features and fixes.** Keep the main repo checkout on `main`.
2. Create a worktree: `git worktree add /tmp/nanoclaw-<feature> -b agent/<feature>`
3. Do all development, commits, and pushes from the worktree.
4. Create a PR from the worktree branch. Never push directly to `main`.
5. After merge, remove the worktree: `git worktree remove /tmp/nanoclaw-<feature>`

This ensures the live service's working tree stays on `main` and auto-update only picks up reviewed, merged changes. Multiple agents can develop concurrently without interfering with each other or the running service.

```bash
npm run dev          # Run with hot reload
npm run build        # Compile TypeScript
./container/build.sh # Rebuild agent container
```

Service management:
```bash
# macOS (launchd)
launchctl load ~/Library/LaunchAgents/com.nanoclaw.plist
launchctl unload ~/Library/LaunchAgents/com.nanoclaw.plist
launchctl kickstart -k gui/$(id -u)/com.nanoclaw  # restart

# Linux (systemd)
systemctl --user start nanoclaw
systemctl --user stop nanoclaw
systemctl --user restart nanoclaw
```

## Troubleshooting

**WhatsApp not connecting after upgrade:** WhatsApp is now a separate channel fork, not bundled in core. Run `/add-whatsapp` (or `git remote add whatsapp https://github.com/qwibitai/nanoclaw-whatsapp.git && git fetch whatsapp main && (git merge whatsapp/main || { git checkout --theirs package-lock.json && git add package-lock.json && git merge --continue; }) && npm run build`) to install it. Existing auth credentials and groups are preserved.

## Container Build Cache

The container buildkit caches the build context aggressively. `--no-cache` alone does NOT invalidate COPY steps — the builder's volume retains stale files. To force a truly clean rebuild, prune the builder then re-run `./container/build.sh`.
