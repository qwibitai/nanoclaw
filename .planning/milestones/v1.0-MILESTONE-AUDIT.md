## Tech Debt & Non-Critical Gaps

### Issue 1: `WEBSOCKET_FILES_PORT` Not Explicitly Passed in main()

**Location:** `src/index.ts:530`
**Severity:** Non-blocking (tech debt)
**Current Behavior:** `new WebSocketChannel(WEBSOCKET_PORT, { onMessage, onChatMetadata })` omits filesPort, uses default from constructor
**Recommendation:** Document implicit wiring or refactor to pass filesPort explicitly for multi-group WS support

### Issue 2: WebSocketServer Binds to All Interfaces

**Location:** `src/channels/websocket.ts:170`
**Severity:** Non-blocking (defense-in-depth)
**Current Behavior:** `new WebSocketServer({ port: this.port })` listens on `0.0.0.0:3001`; file server binds to `127.0.0.1:3002`
**Recommendation:** Add `host: 'localhost'` to WebSocketServer for consistent binding

### Issue 3: `groups/better-work/CLAUDE.md` Missing Attachment Path Documentation

**Location:** `groups/better-work/CLAUDE.md`
**Severity:** Non-blocking (agent guidance)
**Current Behavior:** Generic reference to `/workspace/group/` without specific mention of attachment directories
**Recommendation:** Document `inbox/attachments/` (inbound) and `files/` (outbound) conventions

### Issue 4: File Download Timing Dependency

**Location:** `src/channels/websocket.ts:145`
**Severity:** Non-blocking (known limitation)
**Current Behavior:** File attachment included only if `fs.existsSync()` returns true at sendMessage time
**Recommendation:** Document as known behavior; panel UI should handle both structured and unstructured file links
