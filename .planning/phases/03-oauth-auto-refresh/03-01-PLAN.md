# Plan 03-01: OAuth Module + Container-Runner Integration + Tests

**Phase:** 03 — OAuth Auto-Refresh
**Goal:** Build `src/oauth.ts` with token read/refresh/write logic, wire into `readSecrets()`, add tests.

## Tasks

### Task 1: Create `src/oauth.ts` — credentials file I/O

Create the OAuth module with:

- `OAuthCredentials` interface: `{ accessToken, refreshToken, expiresAt }`
- `CREDENTIALS_PATH` — `path.join(process.cwd(), 'oauth-credentials.json')`
- `readCredentials(): OAuthCredentials | null` — read and parse JSON, return null on missing/corrupt
- `writeCredentials(creds: OAuthCredentials): void` — atomic temp+rename write with `mode: 0o600`
- Export `TOKEN_ENDPOINT`, `CLIENT_ID` as constants

**Files:** `src/oauth.ts` (new)

### Task 2: Add `refreshOAuthToken()` to `src/oauth.ts`

Add the refresh function:

- `refreshOAuthToken(refreshToken: string): Promise<{ accessToken, refreshToken?, expiresIn? }>`
- POST to `https://platform.claude.com/v1/oauth/token`
- Body: `grant_type=refresh_token&refresh_token={token}&client_id={client_id}` (use `URLSearchParams`)
- Content-Type: `application/x-www-form-urlencoded`
- Parse JSON response, throw on non-2xx
- Log refresh attempts and outcomes via `logger`

**Files:** `src/oauth.ts`

### Task 3: Add `getValidToken()` with refresh deduplication

Add the main entry point:

- `getValidToken(): Promise<string>` — exported
- Read credentials file. If missing, fall back to `readEnvFile(['CLAUDE_CODE_OAUTH_TOKEN'])` (migration path)
- If `expiresAt > Date.now() + REFRESH_BUFFER_MS` (5 min), return `accessToken` (fast path)
- If near expiry, refresh. Deduplicate concurrent calls via module-level `refreshPromise`
- On refresh success: persist new credentials (including rotated refresh token if present)
- On refresh failure: log CRITICAL, return stale token as fallback
- `refreshPromise` set to `null` in `finally` block

**Files:** `src/oauth.ts`

### Task 4: Wire `getValidToken()` into `container-runner.ts`

- Change `readSecrets()` to `async readSecrets(): Promise<Record<string, string>>`
- Replace `readEnvFile(['CLAUDE_CODE_OAUTH_TOKEN', ...])` with: call `getValidToken()` for OAuth, `readEnvFile()` for the rest
- Set `CLAUDE_CODE_OAUTH_TOKEN` in the returned secrets from `getValidToken()` result
- Update call site at line ~318: `input.secrets = await readSecrets()`
- This is already inside an async function (`runContainerAgent`), so the change is minimal

**Files:** `src/container-runner.ts`

### Task 5: Write tests for `src/oauth.ts`

Create `src/oauth.test.ts`:

- **readCredentials**: returns parsed credentials from valid file; returns null on missing file; returns null on corrupt JSON
- **writeCredentials**: writes atomic file with correct content; file has 0o600 permissions
- **refreshOAuthToken**: sends correct POST body; parses success response; throws on non-2xx; handles missing refresh_token in response
- **getValidToken**: returns accessToken when not expired; calls refresh when near expiry; deduplicates concurrent refresh calls; falls back to .env when no credentials file; falls back to stale token on refresh failure; persists rotated refresh token

Mock `fetch` globally and `fs` operations where needed. Use the existing test patterns from `container-runner.test.ts`.

**Files:** `src/oauth.test.ts` (new)
