# Plan 03-02: VPS Deployment + Credentials Seed + Verify Refresh

**Phase:** 03 — OAuth Auto-Refresh
**Goal:** Deploy OAuth auto-refresh to VPS, seed credentials file, verify token refreshes automatically, remove static token from `.env`.

## Tasks

### Task 1: Seed `oauth-credentials.json` on VPS

SSH to VPS and create the credentials file from current keychain values:

```bash
ssh craig@5.196.90.31 "sudo -n -u nanoclaw bash -c 'cat > /opt/nanoclaw/oauth-credentials.json << EOF
{
  \"accessToken\": \"<current access token from .env>\",
  \"refreshToken\": \"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\",
  \"expiresAt\": <current expiresAt>
}
EOF
chmod 600 /opt/nanoclaw/oauth-credentials.json'"
```

Verify file exists and has correct permissions (600, owned by nanoclaw).

### Task 2: Deploy code to VPS

```bash
ssh craig@5.196.90.31 "sudo -n -u nanoclaw /opt/nanoclaw/deploy.sh"
ssh craig@5.196.90.31 "sudo -n -u nanoclaw sudo -n systemctl restart nanoclaw"
```

Verify service is active after restart.

### Task 3: Verify OAuth read works

Check NanoClaw logs to confirm it reads from `oauth-credentials.json`:

```bash
ssh craig@5.196.90.31 "sudo -n -u nanoclaw tail -50 /opt/nanoclaw/logs/nanoclaw.log"
```

Send a test message to Holly via Google Chat. Confirm she responds (proving container spawned with valid token).

### Task 4: Remove `CLAUDE_CODE_OAUTH_TOKEN` from `.env`

Once verified working:

```bash
ssh craig@5.196.90.31 "sudo -n -u nanoclaw sed -i 's/^CLAUDE_CODE_OAUTH_TOKEN=/#CLAUDE_CODE_OAUTH_TOKEN=/' /opt/nanoclaw/.env"
ssh craig@5.196.90.31 "sudo -n -u nanoclaw sudo -n systemctl restart nanoclaw"
```

Send another test message to confirm Holly still responds (now reading from credentials file only).

### Task 5: Verify auto-refresh (manual trigger or wait)

Option A (fast): Temporarily set `expiresAt` to `Date.now() + 60000` (1 minute in future) in the credentials file. Wait 2 minutes, send a message. Check logs for refresh activity. Restore to real expiry.

Option B (wait): Leave it running. After ~15 hours, check that Holly is still responding and logs show a successful refresh.

Confirm:
- Logs show "OAuth token refreshed" (or similar)
- `oauth-credentials.json` has new `accessToken` and updated `expiresAt`
- Holly responds after refresh

## Verification

- Holly responds to a Google Chat message after deployment
- Holly responds after `.env` token is commented out
- Logs confirm credentials file is being read
- (Optional) Logs confirm token refresh after forced near-expiry
