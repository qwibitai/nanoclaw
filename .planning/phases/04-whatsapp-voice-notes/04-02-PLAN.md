Add a new `send_audio` MCP tool to the stdio MCP server in `container/agent-runner/src/ipc-mcp-stdio.ts`. Place it after the existing `send_message` tool registration.

```typescript
server.tool(
  'send_audio',
  'Send an audio file as a WhatsApp voice note. The file must exist on disk (e.g., generated by a script or downloaded). The file will be sent as an inline-playable voice note on WhatsApp. For Google Chat, audio is not supported inline — use a Drive link instead.',
  {
    file_path: z
      .string()
      .describe(
        'Absolute path to the audio file inside the container (e.g., /tmp/briefing.ogg). Must be OGG/Opus format for WhatsApp voice notes.',
      ),
    target_jid: z
      .string()
      .optional()
      .describe(
        '(Main group only) JID to send the voice note to. Defaults to the current chat.',
      ),
    mimetype: z
      .string()
      .optional()
      .describe(
        'MIME type of the audio file. Defaults to "audio/ogg; codecs=opus" for WhatsApp voice notes.',
      ),
  },
  async (args) => {
    const targetJid = isMain && args.target_jid ? args.target_jid : chatJid;

    // Verify file exists
    if (!fs.existsSync(args.file_path)) {
      return {
        content: [
          {
            type: 'text' as const,
            text: `Audio file not found: ${args.file_path}`,
          },
        ],
        isError: true,
      };
    }

    // Copy audio file to IPC media directory
    const MEDIA_DIR = path.join(IPC_DIR, 'media');
    fs.mkdirSync(MEDIA_DIR, { recursive: true });
    const audioFilename = `${Date.now()}-${Math.random().toString(36).slice(2, 8)}${path.extname(args.file_path) || '.ogg'}`;
    fs.copyFileSync(args.file_path, path.join(MEDIA_DIR, audioFilename));

    // Write IPC instruction to messages directory
    writeIpcFile(MESSAGES_DIR, {
      type: 'send_audio',
      chatJid: targetJid,
      filePath: audioFilename,
      mimetype: args.mimetype || 'audio/ogg; codecs=opus',
      groupFolder,
      timestamp: new Date().toISOString(),
    });

    return {
      content: [
        {
          type: 'text' as const,
          text: `Voice note queued for sending to ${targetJid}`,
        },
      ],
    };
  },
);
```

Key design decisions:

- The tool copies the audio file to `/workspace/ipc/media/` (not moves) so the original file remains available if the agent needs it.
- File naming uses timestamp + random suffix to avoid collisions.
- The JSON instruction goes to `/workspace/ipc/messages/` (same directory as text messages) so the existing IPC polling picks it up.
- Non-main groups can only send audio to their own chat (same auth model as send_message).
- No WAV→OGG conversion here — the container agent is responsible for generating/converting to OGG before calling this tool. ffmpeg is available in the container for conversion.
