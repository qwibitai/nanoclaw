<!-- src/types.ts — interfaz Channel -->
```typescript
export interface Channel {
  name: string;
  connect(): Promise<void>;
  sendMessage(jid: string, text: string): Promise<void>;
  isConnected(): boolean;
  ownsJid(jid: string): boolean;
  disconnect(): Promise<void>;
  setTyping?(jid: string, isTyping: boolean): Promise<void>;
}

export interface NewMessage {
  id: string;
  chat_jid: string;
  sender: string;
  sender_name: string;
  content: string;
  timestamp: string;
  is_from_me?: boolean;
  is_bot_message?: boolean;
}

export type OnInboundMessage = (chatJid: string, message: NewMessage) => void;

export type OnChatMetadata = (
  chatJid: string,
  timestamp: string,
  name?: string,
  channel?: string,
  isGroup?: boolean,
) => void;
```

<!-- Protocolo de mensajes (CONTEXT.md — decisiones bloqueadas) -->

MENSAJES ENTRANTES del cliente WS:
- `{type:"chat", content:string}` → NewMessage con content = el string literal
- `{type:"system", action:string, payload:object}` → NewMessage con content = `[SYSTEM] {action}: {JSON.stringify(payload)}`
- JSON inválido → enviar `{type:"error", code:"PARSE_ERROR", message:"Invalid JSON"}` al cliente

MENSAJES SALIENTES al cliente WS:
- sendMessage: `{type:"chat", content:text}`
- setTyping: `{type:"system", event:"typing", payload:{isTyping}}`
- Error: `{type:"error", code:"...", message:"..."}`
- Al conectar: `{type:"system", event:"connected", payload:{buffered_count:N}}`
- Al desconectar cliente: enviar `[SYSTEM] client_disconnected: {}` al agente vía opts.onMessage

BUFFER (PROTO-05 — decisiones bloqueadas):
- Máximo 50 mensajes
- Overflow: drop oldest (shift del array)
- Al reconectar: primero `{type:"system", event:"buffered_start", count:N}`, luego los mensajes con timestamp original, luego `{type:"system", event:"buffered_end"}`
- Los mensajes buffereados llevan su timestamp original en el objeto

HEARTBEAT (CONTEXT.md — decisiones bloqueadas):
- Ping nativo WS cada 30 segundos
- Si no hay pong en 10 segundos: terminar conexión como zombie
- Usar ws.ping() nativo (NO mensajes de aplicación)

JID:
- Constante: WS_JID = 'ws:better-work'
- ownsJid(jid): jid.startsWith('ws:')

PITFALL CRÍTICO — onChatMetadata (RESEARCH.md Pitfall 5):
- WebSocketChannel DEBE llamar opts.onChatMetadata(WS_JID, timestamp, 'better-work', 'websocket', false)
- Llamar en connect() cuando el servidor arranca (no solo al primer mensaje)
- Sin esto: FOREIGN KEY constraint failed en SQLite al primer mensaje

    **Estructura requerida:**

    ```typescript
    import crypto from 'node:crypto';
    import { WebSocketServer, WebSocket } from 'ws';
    import type { Channel, NewMessage, OnInboundMessage, OnChatMetadata } from '../types.js';
    import { logger } from '../logger.js';

    const WS_JID = 'ws:better-work';
    const MAX_BUFFER = 50;
    const HEARTBEAT_INTERVAL_MS = 30_000;
    const PONG_TIMEOUT_MS = 10_000;

    export interface WebSocketChannelOpts {
      onMessage: OnInboundMessage;
      onChatMetadata: OnChatMetadata;
    }

    export class WebSocketChannel implements Channel {
      name = 'websocket';
      private wss: WebSocketServer | null = null;
      private client: WebSocket | null = null;
      private buffer: Array<{ content: string; timestamp: string }> = [];
      private heartbeatInterval: ReturnType<typeof setInterval> | null = null;
      private pongTimeout: ReturnType<typeof setTimeout> | null = null;

      constructor(private readonly port: number, private readonly opts: WebSocketChannelOpts) {}
      // ... métodos
    }
    ```

    **connect():**
    - Crea WebSocketServer en this.port
    - Llama opts.onChatMetadata(WS_JID, new Date().toISOString(), 'better-work', 'websocket', false) inmediatamente — CRÍTICO para evitar FK constraint en SQLite
    - En evento 'connection': si ya hay cliente, terminate() el anterior; asignar nuevo cliente
    - Enviar `{type:"system", event:"connected", payload:{buffered_count: this.buffer.length}}` al cliente nuevo
    - Llamar flushBuffer(ws) después del connected event (los buffered siguen al connected)
    - Iniciar heartbeat (ver abajo)
    - En ws 'message': parsear JSON, llamar handleInboundMessage()
    - En ws 'close': clearInterval heartbeat, clearTimeout pongTimeout, this.client = null, notificar al agente con [SYSTEM] client_disconnected
    - En ws 'error': logger.error

    **Heartbeat:**
    - let isAlive = true por cliente
    - ws.on('pong', () => { isAlive = true; if (this.pongTimeout) { clearTimeout(this.pongTimeout); this.pongTimeout = null; } })
    - setInterval cada 30s: si !isAlive → ws.terminate(); else → isAlive = false, ws.ping(), setTimeout 10s para verificar pong
    - Limpiar heartbeatInterval y pongTimeout en ws 'close'

    **handleInboundMessage(ws, raw):**
    - JSON.parse; si falla → ws.send({type:"error", code:"PARSE_ERROR", message:"Invalid JSON"})
    - Si type === 'chat' && content es string → opts.onMessage(WS_JID, {id: crypto.randomUUID(), chat_jid: WS_JID, sender: 'ws:user', sender_name: 'User', content: msg.content, timestamp: new Date().toISOString()})
    - Si type === 'system' → content = `[SYSTEM] ${msg.action}: ${JSON.stringify(msg.payload ?? {})}` → opts.onMessage mismo patrón
    - Otros types: ignorar silenciosamente

    **sendMessage(jid, text):**
    - Si isConnected() → this.client.send(JSON.stringify({type:'chat', content:text}))
    - Si no → bufferMessage(text) (guarda {content:text, timestamp: new Date().toISOString()})

    **bufferMessage(content):**
    - Si this.buffer.length >= MAX_BUFFER → this.buffer.shift() (drop oldest)
    - this.buffer.push({content, timestamp: new Date().toISOString()})

    **flushBuffer(ws):**
    - Si buffer vacío → return
    - ws.send({type:"system", event:"buffered_start", count: buffer.length})
    - Para cada msg en buffer: ws.send({type:"chat", content:msg.content, timestamp:msg.timestamp})
    - ws.send({type:"system", event:"buffered_end"})
    - this.buffer = []

    **isConnected():**
    - return this.client !== null && this.client.readyState === WebSocket.OPEN

    **ownsJid(jid):**
    - return jid.startsWith('ws:')

    **setTyping(jid, isTyping):**
    - this.client?.send(JSON.stringify({type:"system", event:"typing", payload:{isTyping}}))

    **disconnect():**
    - Si heartbeatInterval → clearInterval; si pongTimeout → clearTimeout
    - Si wss → wss.close(); this.wss = null
    - this.client = null

    **Anti-patterns a evitar (RESEARCH.md):**
    - No usar ws.close() en zombies — usar ws.terminate()
    - No lanzar errores en sendMessage cuando no hay cliente — bufferear silenciosamente
    - No reiniciar el servidor en cada desconexión — el wss es persistente
